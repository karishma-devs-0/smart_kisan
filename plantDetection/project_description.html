Project Overview

What Does This Project Do?

This project trains a Convolutional Neural Network (CNN) to look at a photo of a plant leaf and determine whether it's healthy or diseased — and if diseased, exactly which disease it has.

It uses the PlantVillage dataset, which contains tens of thousands of labeled leaf images across 38 categories (e.g., Apple — Black Rot, Tomato — Late Blight, Corn — Healthy).

Analogy

Think of it like studying for an exam. You practice with 80% of the questions (training) and save 20% as a mock test (validation) to see if you actually learned the material — not just memorized answers.

Key Logic images = [f for f in os.listdir(cls_path) if f.lower().endswith(('.jpg', '.jpeg', '.png'))] random.shuffle(images) # Randomize order split_idx = int(len(images) * 0.8) # First 80% → train/ for img in images[:split_idx]: shutil.copy2(src, train_dst) # Remaining 20% → valid/ for img in images[split_idx:]: shutil.copy2(src, valid_dst)

Why random.shuffle? Without it, the split would be alphabetical — images starting with A–S go to train, T–Z go to valid. That could introduce bias if filenames correlate with image properties.

Why copy2 not copy? copy2 preserves file metadata (timestamps). Not critical here, but good practice.

Step 3: Load with ImageDataGenerator

Why resize to 224×224?

Three reasons: (1) MobileNetV2 was trained at this exact size — its internal math expects it. (2) Every image in a batch must be the same dimensions to form a single tensor. (3) A phone photo (4032×3024) has 700× more pixels — resizing keeps memory manageable.

Model Architecture

Transfer Learning — The Core Idea

Instead of training a CNN from scratch (which needs millions of images), we take MobileNetV2 — a model already trained on 1.2 million ImageNet images. It already knows how to detect edges, textures, and shapes.

We remove its original classification head (1000 ImageNet classes) and attach our own (38 plant disease classes). This way, we reuse its learned knowledge and only train the final layers.

Analogy

Imagine hiring a chef who's mastered French cuisine (ImageNet). Instead of teaching them to cook from zero, you just teach them Indian recipes (plant diseases). Their knife skills, heat control, and timing (feature extraction) transfer directly.

Why MobileNetV2?

Layer-by-Layer Architecture

Hover over each layer to learn what it does:

MobileNetV2 Pretrained
155 layers, frozen initially
(7, 7, 1280)
Takes a 224×224×3 image and extracts 1280 feature maps of size 7×7. These features represent high-level patterns (leaf shape, spot textures, color distributions) learned from ImageNet.
GlobalAveragePooling2D
Averages each feature map spatially
(1280,)
Converts each 7×7 feature map into a single number by averaging all 49 values. Output: a 1280-dimensional vector. Much more efficient than Flatten (which would give 62,720 values).
Dropout 30%
Regularization layer
(1280,)
Randomly zeroes out 30% of values during training. Forces the network to not rely on any single feature, reducing overfitting. Disabled during inference (predictions).
Dense (256, ReLU)
Fully connected hidden layer
(256,)
1280 inputs → 256 outputs. ReLU activation: max(0, x) — keeps positive values, zeroes negatives. This layer learns which combinations of MobileNetV2 features matter for plant diseases.
Dropout 30%
Regularization layer
(256,)
Second dropout layer for additional regularization before the final classification.
Dense (38, Softmax) Output
One neuron per disease class
(38,)
256 inputs → 38 outputs. Softmax converts raw scores into probabilities that sum to 1.0. The highest probability is the predicted disease. Example: [0.02, 0.01, 0.92, ...] = 92% confident it's class 2.

Two-Phase Training

Why Two Phases?

Training happens in two stages to protect the pretrained knowledge while still adapting it to our task.

Phase 1 — Feature Extraction

Train the Custom Head Only

MobileNetV2 is completely frozen. Only the Dense + Dropout layers update. The model learns which pretrained features map to which diseases.

10
Max Epochs
0.001
Learning Rate
Phase 1 base_model.trainable = False model.fit(train_generator, epochs=10)
Phase 2 — Fine-Tuning

Unfreeze & Adapt the Last 30 Layers

The last 30 layers of MobileNetV2 are unfrozen. These higher-level layers learn to detect leaf-specific features (spot patterns, discoloration, wilting shapes) instead of generic ImageNet features.

25
Max Epochs
0.0001
Learning Rate (10× lower)
Phase 2 base_model.trainable = True for layer in base_model.layers[:-30]: layer.trainable = False # Keep early layers frozen model.compile(lr=1e-4) # 10× smaller steps! model.fit(train_generator, epochs=25)
Why 10× lower learning rate?

Pretrained weights are already good. Large updates would destroy them. Small updates gently adapt them to leaf images — like fine-tuning a radio dial, not spinning it wildly.

Training Callbacks

Callbacks are automatic actions that run after each epoch. They prevent wasted time, save the best model, and optimize the learning rate.

⏹
EarlyStopping
Monitors val_loss. If it doesn't improve for 5 consecutive epochs, training stops. Rolls back to the best epoch's weights.
Why? After a point, more training = overfitting. The model memorizes training data noise instead of learning real patterns. EarlyStopping catches this exact moment.
EarlyStopping(monitor='val_loss', patience=5, restore_best_weights=True)
⚒
ReduceLROnPlateau
If loss plateaus for 3 epochs, multiplies learning rate by 0.2. Won't go below 0.000001.
How it works: 0.001 → 0.0002 → 0.00004 → ...
Why? When the model is stuck at a plateau, smaller steps help it descend into a finer minimum.
ReduceLROnPlateau(factor=0.2, patience=3, min_lr=1e-6)
💾
ModelCheckpoint
After each epoch, if val_accuracy is the best so far, saves the model to Google Drive.
Why? Training might crash (Colab timeouts), or later epochs might overfit. This guarantees you always have the best version saved.
ModelCheckpoint('best_model.keras', save_best_only=True)

Export & Deployment

What Gets Saved

1

Keras Model .keras

Full model (architecture + weights + optimizer). Can be reloaded for further training or evaluation.

2

Class Labels .json

Maps index numbers to class names: {0: "Apple___Apple_scab", 1: "Apple___Black_rot", ...}. Needed at inference time to convert prediction index to a human-readable name.

3

TFLite Model .tflite

Compressed version with post-training quantization (32-bit floats → 8-bit ints). ~4× smaller, runs on mobile devices and edge hardware.

TFLite Conversion Code

Python converter = tf.lite.TFLiteConverter.from_keras_model(model) converter.optimizations = [tf.lite.Optimize.DEFAULT] # Quantize tflite_model = converter.convert() with open('model.tflite', 'wb') as f: f.write(tflite_model)

Optimize.DEFAULT applies dynamic range quantization — weights are stored as 8-bit integers instead of 32-bit floats. The model activations are still computed in float at runtime, giving a good balance of size reduction and accuracy.

Inference Function

The predict_disease() function takes a single image path and returns the top-3 predictions:

Python img = image.load_img(path, target_size=(224, 224)) img_array = image.img_to_array(img) / 255.0 # Normalize img_array = np.expand_dims(img_array, axis=0) # Add batch dim prediction = model.predict(img_array) # Shape: (1, 38) top3 = np.argsort(prediction[0])[-3:][::-1]

expand_dims adds a batch dimension because the model always expects a batch — even for a single image, the shape must be (1, 224, 224, 3) not (224, 224, 3).

Key Concepts Explained

Epoch

One complete pass through the entire training dataset.

If you have 40,000 images and batch size 32: 1 epoch = 1,250 batches. The model sees every image once. With 10 epochs, it sees every image 10 times — like re-reading a textbook to understand it better.

Learning Rate

Controls how much the model adjusts weights after each batch. Think of it as step size while walking downhill to find the lowest point.

RateStep SizeRisk
0.1 (too high)Huge leapsOvershoots the minimum, never converges
0.001 (good)Moderate stepsReaches minimum efficiently
0.0000001 (too low)Tiny shufflesTakes forever, may get stuck

Formula: new_weight = old_weight - (learning_rate × gradient)

Interactive: Learning Rate Visualizer

Drag the slider to see how different learning rates affect optimization. The ball tries to reach the lowest point of the curve.

0.001

Softmax

The final activation function. Converts raw output scores into probabilities that sum to 1.0.

Example Raw scores: [2.1, 5.7, 0.3, ... ] After softmax: [0.02, 0.92, 0.01, ... ] # sum = 1.0 Prediction: class 1 with 92% confidence

Categorical Crossentropy (Loss Function)

Measures how wrong the model's predicted probabilities are compared to the true label. Formula: -∑ y_true × log(y_pred)

  • If the model predicts 0.92 for the correct class → loss is low (good)
  • If the model predicts 0.02 for the correct class → loss is very high (bad)
  • The optimizer minimizes this value during training

One-Hot Encoding

Each label is a vector of length 38 with a 1 at the true class index and 0 everywhere else.

Example Class 0 (Apple___Apple_scab): [1, 0, 0, 0, ..., 0] Class 2 (Apple___Cedar_rust): [0, 0, 1, 0, ..., 0] Class 37 (Tomato___healthy): [0, 0, 0, 0, ..., 1]

This format works with softmax + crossentropy. np.argmax() converts it back to an integer (e.g., argmax([0,0,1,0,...]) = 2).
